# Known Limitations & Hardening Backlog

> Wire **v0.5.2**. This catalogs findings from an adversarial review of NCP (correctness, safety, robustness, overhead). **These are proposals — none are applied yet.** Each notes whether a fix would change the on-wire contract (`wire-breaking`, needs a version bump + consumer buy-in across Engram/crebain/prisoma) or is internal (`safe`). NCP is a generic, shared contract, so conservatism here is deliberate.


**Summary:** 3 high · 17 medium · 15 low · 3 wire-breaking.


## High — safety / robustness

- **decode: per-column allocations have no cumulative budget — overlapping/duplicate columns enable ~64,000x memory amplification (OOM DoS)** — `bulk.rs:356` · _safety_ · safe
  - _Proposed:_ In BulkBlock::decode, track a running allocation budget and reject when total declared payload exceeds the input size. e.g. before the loop `let mut budget = bytes.len();` and inside the loop after computing data_len: `budget = budget.checked_sub(data_len).ok_or(BulkError::OutOfBounds)?;` (optionally also subtract name_len). Every conforming block lays columns out disjointly so sum(data_len) <= total_len = bytes.len(), so this rejects only overlapping/amplifying blocks and accepts all well-formed ones.
- **Unbounded ttl_ms defeats the CommandWatchdog deadline backstop (fail-OPEN)** — `safety.rs:417` · _safety_ · safe
  - _Proposed:_ Clamp the enforced ttl to a finite documented maximum, e.g. `self.ttl_s = ttl_ms.clamp(0.0, MAX_TTL_MS)/1000.0` where MAX_TTL_MS is a const (or derived from SafetyLimits.command_timeout_ms). Also guard non-finite: a non-finite ttl_ms should map to 0 (immediately stale). Wire still carries ttl_ms unchanged; only local enforcement is bounded.
- **Empty position-channel data bypasses the geofence (treated as origin r=0)** — `safety.rs:293` · _safety_ · safe
  - _Proposed:_ In the `Some(pos)` arm, before computing r: `if pos.data.is_empty() { return self.hold_frame(command); }` (treat like the None arm). Mirror the same emptiness guard at the horizon look-ahead (safety.rs:338).

## Medium

- **Single RPC key per realm prevents per-verb ACL — the shipped ACL lets observer/robot close or step any session** — `zenoh-access-control.json5:81` · _design-gap_ · **wire-breaking**
  - _Proposed:_ Split the privileged verbs onto distinct key-expressions (e.g. `{realm}/rpc/open` vs `{realm}/rpc/admin`) so the ACL can allow clients to open but restrict step/run/close to the commander; or, if the single key must stay, have the RPC handler authorize the caller's proven identity per verb. The key-split is the robust fix but changes the on-wire RPC addressing other peers use.
- **Bulk send/receive paths each perform two full copies of the numeric arrays, undercutting the 'parse-free / low-copy' design** — `bulk.rs:119` · _overhead_ · safe
  - _Proposed:_ Receive: add consuming accessors (e.g. Column::into_f64(self)/into_i64(self)) and an apply that takes the BulkBlock by value, so the matching-width arms move the Vec out instead of cloning. Send: add an encode path that borrows the Observation's slices directly (encode_from(times, values, senders)) to skip the to_bulk_block clone. Optionally make decode_column copy the whole column with copy_from_slice on little-endian targets. No wire-format change.
- **codec encode maps a missing/short sensor channel to rate_range_hz.0 (= value_range minimum), the same 'missing data -> extreme actuation' hazard the decode side was explicitly hardened against** — `codec.rs:128` · _safety_ · safe
  - _Proposed:_ Mirror the decode fix: on a missing/short channel, insert the rate corresponding to the neutral midpoint, e.g. `let neutral = 0.5*(m.value_range.0 + m.value_range.1); rates.insert(m.population.clone(), lerp(neutral, m.value_range.0, m.value_range.1, m.rate_range_hz.0, m.rate_range_hz.1));` (equivalently the midpoint of rate_range_hz), or skip emitting the population so downstream hold-last applies. Changes only the degraded/missing-data path, not the rate-map structure.
- **ncp_version is neither required nor checked on the data plane: an absent version defaults to our own and incompatible-but-parseable sensor/command frames are accepted** — `messages.rs:1219` · _design-gap_ · **wire-breaking**
  - _Proposed:_ Have data-plane decoders call check_version(frame.ncp_version, strict) on each successfully-parsed sensor/command frame and drop on mismatch (reusing the existing diagnose path for the log line), and treat an absent ncp_version as incompatible rather than coercing to ours. At minimum, add 'ncp_version' to required_fields() for every kind so validate() enforces the spec's 'every message carries ncp_version'.
- **seq==0 is an always-accept escape hatch that bypasses the normative anti-replay/anti-stale guarantee — and seq=0 is the wire default** — `resilience.rs:48` · _safety_ · **wire-breaking**
  - _Proposed:_ Restrict the seq==0 escape hatch to the pull/sim service path only. On the streaming action plane, require strictly-positive, strictly-advancing seq: have the plant-side ActionBuffer/CommandWatchdog NOT treat seq==0 as advancing (drop or HOLD), and add 'seq > 0' as a conformance requirement for command_frame on the action plane in §6A. Document seq=0 as 'anti-replay disabled — never use on the action plane.'
- **LinkMonitor seq arithmetic overflows on mixed-sign seqs, silently failing the jam detector open** — `resilience.rs:204` · _safety_ · safe
  - _Proposed:_ Use saturating subtraction: line 204 `let missed = seq.saturating_sub(e);` and line 253 `let span = exp.saturating_sub(first).max(1);`. `missed` then stays non-negative so the observe loop, missing-set fill, and saturating_add all behave; the burst detector trips on the huge gap as intended.
- **Free-text channel units are never validated against the SI units SafetyLimits assumes; the speed clamp and geofence silently mis-scale on a unit mismatch** — `safety.rs:369` · _safety_ · safe
  - _Proposed:_ At SafetyGovernor construction (with_channels/from_capabilities) require the negotiated ChannelSpec.unit of the velocity/position channels to equal the canonical SI unit the limit is expressed in (m/s, m); on mismatch or absent unit when a limit is set, start config_fail_closed=true (the same latched HOLD + safety_ok=false path already used for an undeclared channel). Spec-wise, pin a small canonical-unit vocabulary for safety-relevant channels in NEURO_CYBERNETIC_PROTOCOL.md rather than leaving 'unit' free text.
- **Channel arity is never validated against ChannelKind/size; a truncated vec3 position frame makes the geofence under-report distance and fail to ESTOP** — `safety.rs:293` · _safety_ · safe
  - _Proposed:_ Validate channel arity against the negotiated ChannelSpec before enforcement: in SafetyGovernor, when the geofence/velocity channel's ChannelKind is vec3 (or its size is set), require pos.data.len()/vel.data.len() to equal that width, else fail closed (HOLD for velocity, ESTOP/HOLD for the un-evaluable geofence) instead of computing a magnitude over a short vector. Optionally surface a per-kind arity check in validate() for sensor_frame/command_frame.
- **CommandWatchdog fails OPEN on a non-finite (+Inf) ttl_ms — a single command permanently disables the deadline backstop** — `safety.rs:432` · _safety_ · safe
  - _Proposed:_ Sanitize in on_command: `self.ttl_s = if ttl_ms.is_finite() { ttl_ms.max(0.0) / 1000.0 } else { 0.0 };` (a non-finite ttl is treated as immediately stale), OR add `|| !self.ttl_s.is_finite()` to the should_hold disjunction. Local computation only; no wire change.
- **A NaN safety limit silently DISABLES enforcement (speed clamp / geofence) instead of failing closed** — `safety.rs:307` · _safety_ · safe
  - _Proposed:_ Treat a configured-but-NaN limit as fail-closed. Minimal: at the top of the post-stale section of govern, `if self.limits.max_speed_mps.is_some_and(f64::is_nan) || self.limits.geofence_radius_m.is_some_and(f64::is_nan) { return self.hold_frame(command); }`. Better: detect NaN in detect_misconfig so it becomes a latched config_fail_closed (safety_ok=false). No wire change.
- **Staleness and ttl checks fail OPEN on a backward clock step (non-monotonic now_s / cross-clock last_sensor_s)** — `safety.rs:272` · _safety_ · safe
  - _Proposed:_ Treat negative elapsed as stale/expired: add `|| now_s < last` to the govern disjunction (line 272 area) and `|| now_s < t` to should_hold (line 433), and/or document that callers MUST pass a monotonic clock. No wire change.
- **NaN/negative SafetyLimits silently disable geofence and speed enforcement** — `safety.rs:112` · _safety_ · safe
  - _Proposed:_ In detect_misconfig treat `Some(x)` where `!x.is_finite() || x < 0.0` as bad (fail closed), and in govern replace `radius > 0.0` / `max_speed > 0.0` with explicit `x.is_finite() && x > 0.0`, taking the fail-closed branch when a limit is set but non-finite. Or validate/reject at SafetyGovernor construction.
- **Gateway bridge timeout (30 s) exceeds Zenoh's default query timeout (~10 s) — slow-but-alive backend yields a spurious 'no reply' to the client** — `main.rs:29` · _correctness_ · safe
  - _Proposed:_ Make the bridge timeout strictly less than the effective Zenoh query timeout (e.g. drop the 30 s read/write timeouts to ~8 s), or raise `queries_default_timeout` in the loaded config above the bridge timeout, or set an explicit `.timeout(..)` on the get in request(). Align the two so a successful backend never out-lives the query window.
- **RPC queryable handler runs serially in the recv loop — one slow/hung backend stalls all control-plane RPC** — `lib.rs:319` · _robustness_ · safe
  - _Proposed:_ Clone the `Arc<handler>` and `tokio::spawn` a task per received query (do the handler call + `query.reply().await` inside it) so concurrent RPCs are served in parallel; the recv loop then only dispatches. Keeps the queryable single-key on the wire.
- **Subscriptions are never removed — per-session subscribe on a long-lived bus leaks handles and keeps firing callbacks for closed sessions** — `lib.rs:476` · _design-gap_ · safe
  - _Proposed:_ Return an opaque guard from subscribe()/subscribe_* (e.g. wrap the Subscriber in a handle whose Drop undeclares it), or add an `unsubscribe`/`close_session_subs(session_id)` that removes the matching entries from `subs`. Callers can then scope subscriptions to a session's lifetime. API change only, not wire.
- **close() unconditionally closes the underlying session, including a session borrowed via from_session** — `lib.rs:484` · _robustness_ · safe
  - _Proposed:_ Track whether the bus owns the session (e.g. an `owned: bool` set true only by with_config/open*, false by from_session) and make close() a no-op (or undeclare-only) when not owned; or document that from_session callers must never call close().
- **latest_sensor() deep-clones the entire SensorFrame under a Mutex on every control tick** — `lib.rs:542` · _overhead_ · safe
  - _Proposed:_ Store the latest frame as an `Arc<SensorFrame>` (e.g. `arc_swap::ArcSwapOption<SensorFrame>`, or minimally `Mutex<Option<Arc<SensorFrame>>>`) so the writer stores an Arc and the reader clones only a pointer. Change `ControlTransport::latest_sensor` to return `Option<Arc<SensorFrame>>`; the controller already takes `Option<&SensorFrame>`, so `arc.as_deref()` works at the call site. Eliminates the per-tick deep clone and shrinks/eliminates the lock. Trait-API change, not a wire change.

## Low / micro-optimizations

- **encode silently truncates n_cols/total_len/offsets via unchecked `as u16`/`as u32` casts, producing corrupt blocks instead of erroring** — `bulk.rs:281` · _robustness_ · safe
  - _Proposed:_ Add debug_assert!s (`debug_assert!(n_cols <= u16::MAX as usize)`, `debug_assert!(total_len <= u32::MAX as usize)`, etc.), or provide a fallible `try_encode() -> Result<Vec<u8>, BulkError>` using `u16::try_from`/`u32::try_from` that returns BulkError::Overflow, keeping the infallible encode() as a thin wrapper. No wire-format change.
- **Realm string is interpolated into every key without validation — a wildcard/empty/trailing-slash realm silently widens or breaks the keyspace** — `keys.rs:53` · _robustness_ · safe
  - _Proposed:_ Add a `valid_realm()` check (non-empty; each `/`-separated segment passes valid_id_segment; no `*`/`$`/`#`/`?`/whitespace; no leading/trailing/double slash) and apply it in Keys::new and in the gateway when reading NCP_REALM, returning an error instead of building corrupt keys.
- **Per-publish key strings are rebuilt via nested format! on every frame (plant publishes every tick)** — `keys.rs:64` · _overhead_ · safe
  - _Proposed:_ Cache the per-session, per-plane key strings once (build `keys.sensor(id)` / `keys.command(id)` / `keys.observation(id)` at transport construction, e.g. in `ZenohControlTransport::new` and the plant-side publisher, and store the `String`s) and reuse them in the hot `put` calls instead of recomputing. Non-wire.
- **Keys builders panic (assert!) on an invalid id — process abort instead of fail-safe reject** — `keys.rs:65` · _robustness_ · safe
  - _Proposed:_ Provide fallible builders (e.g. `try_session`/`try_sensor` returning Result<String, KeyError>, or make the existing builders return Result) and use them on any wire-facing path (bus.rs). Reserve `assert!`/panic for clearly-internal, already-validated callers.
- **The per-frame timestamp field `t` has no specified unit or clock domain on any message** — `messages.rs:770` · _design-gap_ · safe
  - _Proposed:_ Pin t's contract in proto/ncp.proto and NEURO_CYBERNETIC_PROTOCOL.md — recommend 'seconds, sender's monotonic clock, not comparable across peers; use seq for correlation and the plant's own clock for liveness.' Or drop t in favor of seq + sim_time_ms if no consumer needs it.
- **Hot action/perception planes are JSON-only; constant string fields (ncp_version/kind/frame_id) are re-encoded every frame** — `messages.rs:794` · _design-gap_ · safe
  - _Proposed:_ Do NOT drop the fields in place (that breaks the self-describing wire, `validate()`, and `diagnose_version`). Instead add an OPTIONAL negotiated binary control-frame codec for the action/perception planes, mirroring how BulkBlock was added additively for observations (peers advertise support in the capabilities handshake; JSON stays the always-available default). That keeps existing JSON peers working (breaksWire=false for the additive path) while giving high-rate deployments a low-allocation, smaller-byte hot frame. Removing/shortening the fields on the existing JSON wire would be breaksWire=true.
- **max_horizon_len returns usize::MAX for non-finite/huge ttl_ms and is only advisory** — `resilience.rs:110` · _robustness_ · safe
  - _Proposed:_ Require finite inputs and clamp the result, e.g. `if !ttl_ms.is_finite() || !horizon_dt_ms.is_finite() || horizon_dt_ms <= 0.0 { return 0; }` then cap to a sane ceiling (e.g. `.min(SOME_MAX)`). Optionally truncate the horizon to this cap inside ActionBuffer::on_command so the documented invariant is enforced plant-side, not just relied upon from senders.
- **Geofence uses full Euclidean norm over all position elements about a hardcoded origin (counts altitude; no configurable center)** — `safety.rs:293` · _design-gap_ · safe
  - _Proposed:_ Document the fence explicitly as a sphere about the frame origin, or restrict the norm to the horizontal (xy) components and/or add a configurable center and per-axis bounds to SafetyLimits. The documentation/horizontal-norm change is local; adding center/ceiling fields to SafetyLimits would extend the wire schema (breaksWire there).
- **SafetyGovernor::govern deep-clones the whole CommandFrame every active tick** — `safety.rs:305` · _overhead_ · safe
  - _Proposed:_ Change the signature to `govern(&mut self, command: CommandFrame, ...) -> CommandFrame` (take by value). The active path then mutates `command` in place and returns it (no clone); the HOLD/ESTOP branches still construct fresh frames from `command.t`/`command.seq`/`zeroed_channels(&command)`. Update the loop to pass `cmd` by value. Removes a per-tick deep clone proportional to channel + horizon size. Non-wire.
- **Inline callback subscriber on the lossy perception plane can head-of-line-block the safety-critical action plane on the same session** — `lib.rs:466` · _robustness_ · safe
  - _Proposed:_ Offer a bounded handler variant for the perception plane that drops instead of back-pressuring (Zenoh `RingChannel`/`FifoChannel` with a small bound, drained on a dedicated task), so a slow perception consumer sheds frames rather than blocking the link that also carries action/RPC. Keep the inline callback for control/observation. Local handler choice; no wire change.
- **serve_rpc cannot be stopped without closing the session, and its doc claims a returned task that does not exist** — `lib.rs:318` · _api_ · safe
  - _Proposed:_ Return the JoinHandle (or an AbortHandle / a small guard whose Drop aborts the task) from serve_rpc, and fix the doc to match; consumers can then stop serving without tearing down the session.
- **Action-plane publish errors are silently swallowed in the streaming control transport** — `lib.rs:538` · _robustness_ · safe
  - _Proposed:_ On Err from publish_command, increment a counter or emit a throttled diagnostic (as the sensor-decode path already does at lib.rs:513-516) so dropped/failed command publishes are observable, while keeping the send itself non-blocking.
- **Zenoh subscribe callback forces a String + Vec<u8> heap allocation on every received frame (every sensor/command tick)** — `lib.rs:468` · _overhead_ · safe
  - _Proposed:_ Change the subscribe callback bound from `Fn(String, Vec<u8>) + Send + Sync + 'static` to `Fn(&str, &[u8]) + Send + Sync + 'static` (matching `ncp_core::SubCallback`) and pass `sample.key_expr().as_str()` and `sample.payload().to_bytes().as_ref()` by reference; callbacks that need to retain the data copy explicitly. Removes 2 allocations/frame on every plane. Source-API change for downstream callbacks (crebain/prisoma) but the wire bytes are unchanged.
- **send_command allocates per command on the latency-critical action plane (Keys/String clone + spawn)** — `lib.rs:530` · _overhead_ · safe
  - _Proposed:_ (1) Make `Keys.realm: Arc<str>` so `bus.clone()` is refcount-only (no per-tick String alloc). (2) Store `session_id` as `Arc<str>` in the transport so its clone is refcount-only. (3) Replace per-command `tokio::spawn` with one long-lived publisher task fed by a `tokio::sync::watch` (latest-wins) or bounded mpsc, so each command is a `try_send` of the serialized bytes rather than a task allocation + schedule. Non-wire.
- **ZenohBus::put re-copies the payload via to_vec() even when the caller already owns the serialized Vec** — `lib.rs:441` · _overhead_ · safe
  - _Proposed:_ Add a `put_owned(&self, key: &str, payload: Vec<u8>, plane: Plane)` (or make `put` generic over `impl Into<zenoh::bytes::ZBytes>`) that moves the owned buffer into `ZBytes` with no copy, and route `send_command`/`publish_command`/RPC through it. Saves one alloc+copy of the full frame per publish on the owned-buffer paths. Non-wire.
